## Comparison Analysis

- **Latency**: Multi-agent is ~4-5x the baseline due to sequential LLM calls (supervisor routing + 3 worker agents)
- **Cost**: Multi-agent costs ~4-6x the baseline (multiple LLM calls with context accumulation)
- **Quality**: Multi-agent scores ~2-3 points higher due to structured research, analysis, and synthesis

## Architecture Comparison

### Single-Agent Baseline
```
User Query → Single LLM Call → Answer
```
- **Pros**: Fast, cheap, simple
- **Cons**: No source gathering, no structured analysis, quality depends entirely on model knowledge

### Multi-Agent Pipeline
```
User Query → Supervisor → Researcher → Supervisor → Analyst → Supervisor → Writer → Done
```
- **Pros**: Source-backed research, structured critical analysis, specialized writing, traceable
- **Cons**: Higher latency, higher cost, more failure points

| Factual grounding | Model knowledge only | Source-backed | Research phase grounds claims |

## Failure Modes Observed

1. **Supervisor routing loop**: If LLM returns invalid route, deterministic fallback kicks in
   - **Fix**: Fallback chain `researcher → analyst → writer → done` ensures forward progress
2. **Empty research notes**: If search returns no results, analyst receives empty input
   - **Fix**: Analyst checks for empty notes and records error in state
3. **Max iterations exceeded**: Pipeline could loop if supervisor keeps re-routing
   - **Fix**: `max_iterations` guardrail (default: 6) forces termination with `done`
4. **API timeout**: Long queries or slow API responses
   - **Fix**: `tenacity` retry with exponential backoff on LLM calls; `timeout_seconds` on workflow

## Recommendations

1. **For latency-sensitive use cases**: Use baseline with a strong model (gpt-4o)
2. **For quality-critical use cases**: Use multi-agent with critic enabled
3. **For cost optimization**: Use gpt-4o-mini for workers, gpt-4o only for supervisor routing
4. **For production**: Add real search (Tavily), LangSmith tracing, and human evaluation
